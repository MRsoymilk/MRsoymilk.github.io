Linux下unzip的乱码问题
  • categories

  • >

通常是因为字符编码不一致导致

Windows 下的 ZIP 文件的默认编码为 CP437 或 GBK

Linux 系统默认使用 UTF-8 编码

使用 unzip 的-O 指定编码

unzip -O CP936 your_file.zip

使用 7z

7z x your_file.zip

使用 iconv

convmv -f GBK -t UTF-8 --notest --recursive your_folder
